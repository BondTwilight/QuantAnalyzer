# 量化策略分析网站 — 架构设计方案

## 🎯 项目目标

搭建一个本地量化策略分析平台，每天自动运行策略回测，通过专业量化指标对比各策略的优劣、可行性、数据支撑度。

## 📋 核心需求

1. **策略库管理**：内置多套经典量化策略（动量、均值回归、多因子、轮动等）
2. **自动回测引擎**：每天自动触发全策略回测
3. **专业指标评估**：年化收益率、最大回撤、夏普比率、Sortino比率、Calmar比率、胜率、盈亏比、信息比率等
4. **策略对比仪表盘**：可视化对比各策略表现
5. **数据来源**：A股市场数据（AKShare 免费获取）

## 🛠️ 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| **Web框架** | Streamlit | Python原生，50行代码可出页面，数据可视化友好 |
| **回测引擎** | Backtrader | 事件驱动，学习曲线低，功能全面，A股支持好 |
| **数据源** | AKShare | 免费无需API Key，98个标准化A股接口 |
| **性能分析** | empyrical | 专业的量化指标计算库（夏普比率、最大回撤等） |
| **可视化** | Plotly + Streamlit | 交互式图表，支持缩放、悬浮数据 |
| **数据库** | SQLite | 轻量级，无需安装，本地存储回测结果 |
| **任务调度** | APScheduler | Python原生调度库，支持cron表达式 |
| **PDF报告** | FPDF | 生成策略分析PDF报告 |

## 📐 内置策略清单

### 第一期（5个核心策略）
1. **双均线交叉策略** (MA Cross) — 经典趋势跟踪
2. **RSI超买超卖策略** — 均值回归
3. **多因子选股策略** — 价值+成长+动量因子
4. **行业轮动策略** — 基于动量的行业配置
5. **动量策略** — 价格动量 + 成交量确认

### 第二期（扩展策略）
6. 布林带策略
7. MACD策略
8. 海龟交易法则
9. 配对交易策略
10. 自定义策略（用户可配置参数）

## 📊 核心评估指标

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| **年化收益率** | 策略年化回报 | (1+总收益)^(252/交易天数) - 1 |
| **最大回撤** | 最大亏损幅度 | 峰值到谷底的最大跌幅 |
| **夏普比率** | 风险调整后收益 | (年化收益-无风险利率) / 年化波动率 |
| **Sortino比率** | 下行风险调整收益 | 只考虑下行波动 |
| **Calmar比率** | 回撤调整收益 | 年化收益 / 最大回撤 |
| **胜率** | 盈利交易占比 | 盈利次数 / 总交易次数 |
| **盈亏比** | 平均盈利/平均亏损 | 平均盈利交易收益 / 平均亏损交易收益 |
| **信息比率** | 相对基准超额 | 超额收益的稳定性 |
| **换手率** | 交易频率 | 年交易次数 |
| **Beta** | 市场敏感度 | 策略收益 vs 基准收益的回归斜率 |

## ⚙️ 自动化调度方案

```
每日执行流程（交易日 15:30 触发）:
┌─ 1. 数据更新 ─── 从AKShare拉取最新行情数据
├─ 2. 策略回测 ─── 运行所有策略回测（增量更新）
├─ 3. 指标计算 ─── 计算全部量化指标
├─ 4. 结果存储 ─── 写入SQLite数据库
├─ 5. 报告生成 ─── 生成当日分析报告
└─ 6. 异常检测 ─── 检测策略失效信号，发出预警
```

## ⚠️ 关于果仁网的说明

果仁网**没有公开API**，官方明确禁止爬虫获取数据且有封号风险。因此采用以下替代方案：

1. **策略逻辑复现**：果仁网上的经典策略（动量轮动、多因子等）用Python代码复现
2. **数据自行获取**：使用AKShare获取A股行情数据，替代果仁的数据源
3. **独立回测引擎**：使用Backtrader自建回测引擎，完全脱离果仁平台依赖

> 如果你一定要接入果仁网数据，唯一合法途径是在果仁网付费订阅策略后，手动导出数据导入本系统。
